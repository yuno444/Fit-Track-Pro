# VERSION 1 scope: must-haves and deferred work

## Must-have for v1

1. Tab navigation — Dashboard, Workouts, Recipes (meals), Profile.
2. Profile — Edit and persist user profile
3. Workouts — Log workout sessions (date, duration, optional category/name, calories burned as entered or estimated), list and edit past sessions.
4. Recipes / meals — Food library with calories per serving; build custom meals from ingredients, log meals so they contribute to daily calorie intake.
5. Dashboard — Weekly summary: workouts that are logged, calories burned, average per workout, total minutes of exercise; 7-day chart for calories burned (Swift Charts).

## Success criteria

- User can complete a full loop: set profile → log meals → log workouts → see dashboard update from stored data without crashing on restart.

# VERSION 2 scope: must-haves and deferred work

## Must-have for v2

1. Create Workout Plan — Generate a workout plan for user based on goals, equipment, rest, etc.
2. Discover Recipes — Let user add ingredients they have to "My pantry", allowing them to browse meals based off recipes they have. Also lets users save recipes that they can reuse.
3. Dashboard — Graph user metrics visually. After users log their meals/metrics (e.g. weight), it will show charts that show their weight trend, nutrition for the week, and training volume.

## Success criteria

- User can use all buttons and browse through UI  with no bugs. Metrics are graphed correctly based on user input. Recipes from "Discover recipes" are properly sorted and shows the correct meals.
