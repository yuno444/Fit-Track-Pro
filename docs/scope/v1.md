# V1 scope: must-haves and deferred work

## Must-have for v1

1. Tab navigation — Dashboard, Workouts, Recipes (meals), Profile.
2. Profile — Edit and persist user profile (fields in [user-profile.md](../domain/user-profile.md)); **Save** action persists to SwiftData.
3. Workouts — Log workout sessions (date, duration, optional category/name, calories burned as entered or estimated); list and edit past sessions.
4. Recipes / meals — Food library with calories per serving; build custom meals from ingredients; log meals so they contribute to daily calorie intake.
5. Dashboard — Weekly aggregates: workouts count, calories burned (from workouts), average per workout, total minutes; 7-day chart for calories burned (Swift Charts).

## Things that will not be in V1 (current prototype)

- IOS Health App — Read/write active energy or workouts from Health; add after core flows are stable.
- Cloud sync — iCloud or custom API; local-only for v1 ([persistence.md](../architecture/persistence.md)).
- Push notifications — Reminders for logging; optional later.
- Social / accounts — Multi-user server auth; single-device profile is enough for v1.

## Success criteria

- User can complete a full loop: set profile → log meals → log workouts → see dashboard update from stored data without crashing on restart.
